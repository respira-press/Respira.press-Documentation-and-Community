Inhale: MCP Abilities is a small WordPress plugin that gives site administrators a settings page for choosing which registered abilities are visible to the default MCP server. It is a free utility built by Respira.

## What Inhale: MCP Abilities does

- Lists every ability registered with WordPress's Abilities API across all active plugins and themes.
- Lets you choose which abilities are exposed to the default MCP server via simple checkboxes.
- Surfaces annotation metadata (`read-only`, `destructive`, `idempotent`) on each ability so you can make informed decisions.
- Asks for a single browser confirmation when you inhale an ability marked destructive.
- Shows the endpoint your MCP client should connect to, with guides for the common clients.

## What Inhale: MCP Abilities doesn't do

- Inhale: MCP Abilities does not run MCP servers, transports, or authentication. Those are handled by the [WordPress MCP Adapter](https://github.com/WordPress/mcp-adapter), which Inhale: MCP Abilities extends.
- Inhale: MCP Abilities does not register any abilities of its own. It only toggles visibility of abilities other plugins have registered.
- Inhale: MCP Abilities does not phone home, collect telemetry, or make external network requests.

## How Inhale: MCP Abilities fits in the Respira ecosystem

Inhale: MCP Abilities is one of three layers Respira ships for WordPress:

| Layer | What it is | Where it lives |
|---|---|---|
| **Inhale: MCP Abilities** | Free utility, choose which abilities are MCP-visible | Settings → Inhale: MCP Abilities |
| **Respira for WordPress** | Paid plugin, safety layer for AI-driven edits across 12 page builders | Respira menu in wp-admin |
| **respira.press** | Marketing site and these docs | Public |

You can run Inhale: MCP Abilities without ever installing Respira for WordPress. The two products are separate; Inhale: MCP Abilities is a community contribution.
